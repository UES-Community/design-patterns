## Propósito

El patrón **Singleton** restringe la instanciación de una clase a un único objeto. Es útil cuando exactamente un objeto debe coordinar acciones a través del sistema: configuración global, loggers, pools de conexión a base de datos.

## Problema

Necesitas asegurarte de que una clase solo tenga **una instancia** y que esa instancia sea fácilmente accesible desde cualquier parte del código. Crear instancias múltiples podría causar comportamientos inconsistentes o duplicar recursos costosos.

## Solución

1. Hacer el constructor **privado** para prevenir instanciación directa.
2. Crear un método estático que actúe como constructor y almacene la instancia en un campo estático.
3. En llamadas subsecuentes, devolver la instancia ya existente.

## Estructura

```
Singleton
├── -instance: Singleton  (estático, privado)
├── -Singleton()           (constructor privado)
└── +getInstance(): Singleton  (estático, público)
```

## Cuándo usar

- Cuando necesitas exactamente una instancia de una clase que sea accesible globalmente.
- Cuando la instancia debe ser extensible por subclases y los clientes deben poder usar la instancia extendida sin modificar su código.

> **Advertencia:** El Singleton es considerado un antipatrón en muchos contextos modernos porque introduce acoplamiento global y dificulta el testing. Considera usar **Inyección de Dependencias** como alternativa.

## Ejemplos de uso real

- Logger de aplicación
- Pool de conexiones a base de datos
- Caché de configuración
- Gestor de archivos del sistema operativo

```typescript
// TypeScript — Thread-safe con inicialización lazy
class DatabaseConnection {
  private static instance: DatabaseConnection | null = null
  private connectionCount = 0

  private constructor(private readonly url: string) {
    console.log(`Connecting to ${url}`)
  }

  static getInstance(url = 'postgres://localhost/app'): DatabaseConnection {
    if (!DatabaseConnection.instance) {
      DatabaseConnection.instance = new DatabaseConnection(url)
    }
    return DatabaseConnection.instance
  }

  query(sql: string): string {
    this.connectionCount++
    return `[Query #${this.connectionCount}] ${sql}`
  }
}

// Uso
const db1 = DatabaseConnection.getInstance()
const db2 = DatabaseConnection.getInstance()

console.log(db1 === db2) // true — misma instancia
console.log(db1.query('SELECT * FROM users'))
```

```python
# Python — usando __new__ para control de instanciación
class DatabaseConnection:
    _instance: 'DatabaseConnection | None' = None

    def __new__(cls, url: str = 'postgres://localhost/app'):
        if cls._instance is None:
            instance = super().__new__(cls)
            instance.url = url
            instance._connection_count = 0
            print(f'Connecting to {url}')
            cls._instance = instance
        return cls._instance

    def query(self, sql: str) -> str:
        self._connection_count += 1
        return f'[Query #{self._connection_count}] {sql}'


# Uso
db1 = DatabaseConnection()
db2 = DatabaseConnection()

print(db1 is db2)          # True — misma instancia
print(db1.query('SELECT * FROM users'))
```

```go
// Go — usando sync.Once para thread-safety garantizada
package singleton

import "sync"

type DatabaseConnection struct {
    url             string
    connectionCount int
}

var (
    instance *DatabaseConnection
    once     sync.Once
)

func GetInstance(url string) *DatabaseConnection {
    once.Do(func() {
        instance = &DatabaseConnection{url: url}
        println("Connecting to " + url)
    })
    return instance
}

func (d *DatabaseConnection) Query(sql string) string {
    d.connectionCount++
    return fmt.Sprintf("[Query #%d] %s", d.connectionCount, sql)
}
```

```java
// Java — Double-Checked Locking para entornos multi-hilo
public class DatabaseConnection {
    private volatile static DatabaseConnection instance;
    private final String url;
    private int connectionCount = 0;

    private DatabaseConnection(String url) {
        this.url = url;
        System.out.println("Connecting to " + url);
    }

    public static DatabaseConnection getInstance(String url) {
        if (instance == null) {
            synchronized (DatabaseConnection.class) {
                if (instance == null) {
                    instance = new DatabaseConnection(url);
                }
            }
        }
        return instance;
    }

    public String query(String sql) {
        connectionCount++;
        return "[Query #" + connectionCount + "] " + sql;
    }
}
```

## Ventajas y desventajas

| | Descripción |
|---|---|
| **Ventaja** | Garantía de instancia única |
| **Ventaja** | Punto de acceso global bien definido |
| **Ventaja** | Inicialización lazy (solo cuando se necesita) |
| **Desventaja** | Viola el Principio de Responsabilidad Única |
| **Desventaja** | Dificulta el unit testing |
| **Desventaja** | Requiere tratamiento especial en entornos multi-hilo |

## Patrones relacionados

- **Abstract Factory**, **Builder**, **Prototype** — pueden implementarse como Singleton
- **Facade** — frecuentemente se implementa como Singleton
