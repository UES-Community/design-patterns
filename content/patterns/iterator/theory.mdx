## Objetivo

El **Iterator** abstrae la forma en que se recorre una colección. El cliente no necesita saber si la colección es un array, un árbol, un grafo o una base de datos — solo llama a `next()`.

### Cuándo usar

- Cuando quieres ocultar la implementación interna de una colección.
- Cuando quieres soportar múltiples formas de recorrer la misma colección.
- Cuando quieres proporcionar una interfaz uniforme para recorrer diferentes tipos de colecciones.
